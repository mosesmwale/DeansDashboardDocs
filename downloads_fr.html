Téléchargements

DHIS2 est un logiciel open source sous licence libérale BSD dont l'installation et l'utilisation sont gratuites.

Vous pouvez dès à présent obtenir le logiciel DHIS2 offert comme un service, hormis quelques aspects comme les fichiers de sauvegarde, les dispositifs de sécurité et la connexion haut débit.

DHIS 2 Live

Ce paquet est extrêmement facile à installer et pratique à utiliser dans la mesure où il contient tout ce dont vous avez besoin pour lancer DHIS2, notamment le SIG, les rapports et les tableaux. Il est basé sur un conteneur de servlet Jetty et une base de données H2, tous deux intégrés. Ouvrez simplement l'archive, lancez le fichier exécutable et vous serez prêts à démarrer. Pour ce qui est de la production, nous vous suggérons d'utiliser PostgreSQL comme DBMS. L'identifiant pour démarrer une session sur la base de données vide est admin / district.

Fichier WAR

Le fichier WAR nécessite que vous installiez un conteneur de servlet Java (de type Tomcat ou Jetty) et une base de données correspondantes (PostgreSQL, MySQL et H2 sont acceptées), et est recommandé pour les paramétrages et les environnements de serveurs avec un volume de données et un trafic importants. La dernière version est maintenue à jour avec une correction des anomalies et de légères améliorations. Vous pouvez toujours obtenir la dernière version la plus stable à stable.dhis2.org. Consultez le guide d'installation pour Ubuntu Linux ici. Pour connaître les dernières nouveautés, consultez le serveur d'intégration continu. Les fichiers WAR sont copiés à partir duserveur d'intégration continu serveur d'intégration continu où vous pouvez trouver le nombre de révisions et le temps de création.

Client mobile Java

Le client mobile DHIS2 fonctionne sur les téléphones mobiles équipés avec Java. La solution s'articule autour d'une connexion des données disponibles également connue sous les appellations GPRS, Edge ou 3G, à travers laquelle il communique avec le serveur DHIS2 rendu disponible au public sur internet.

Deux applications clients séparées sont disponibles : le rapporteur au niveau de l'établissement et l'outil de suivi programmatique. Le client au niveau de l'établissement sert au rapport des données régulières à ce niveau, tandis que l'outil de suivi programmatique est conçu pour le suivi et le rapport concernant les services programmatiques individuels dispensés aux bénéficiaires et intégré comme composante de DHIS2.

Application mobile Android

La capture de données pour l'application Android DHIS2 peut servir à la capture de données et à la gestion des profils. Les données peuvent être sauvegardées hors connexion et être téléchargées vers le serveur lorsqu'une connexion est disponible. Vous pouvez l'obtenir directement par le biais de Google Play

Système d'inventaire de l'équipement pour le maintien de la chaîne de froid (CCEI)

Le système CCEI a été conçu en plus de la plateforme DHIS2 pour permettre aux programmes nationaux de vaccination de recueillir, suivre, évaluer et analyser les données d'inventaire concernant l'équipement en s'appuyant sur l'ensemble des fonctionnalités de DHIS2. Le système CCEI peut être adapté en fonction des besoins pour suivre d'autres types d'équipements d'une importance capitale pour les programmes de santé, notamment les moyens de transport, les appareils de diagnostic ou l'équipement médical.

Vous pouvez télécharger le fichier WAR CCEI le plus récent à partir du lien ci-dessous. Pour un démarrage rapide, vous pouvez également télécharger le fichier démo de sauvegarde postgres. Le CCEI est une ressource logicielle open source unique et nous encourageons les utilisateurs intéressés à examiner le guide d'utilisation du système pour les données standards et à participer au groupe Launchpad CCEI.

Échantillon de données

Lorsque vous configurez votre système, il est utile d'avoir accès à une base comprenant un échantillon de données. Cette base contient des données issues de la mise en œuvre de DHIS2 en Sierra Leone. Veuillez noter que le fichier de la base de données H2 nécessite des privilèges d'écriture sur tout système d'exploitation. Le fichier PostgreSQL doit être décompressé et peut-être importé par le biais de la fonction restore de pgAdmin ou avec psql -d dbname -U username -f demo.sql
